نظام حجز العيادات (Booking System) - الواجهة الخلفية (Backend)

هذا المشروع عبارة عن واجهة خلفية (Backend) قوية ومتكاملة لإنشاء نظام حجز عيادات (Clinic Booking System) مبني باستخدام Node.js, Express, و MongoDB.

يتميز المشروع ببنية تحتية قوية قابلة للتطوير، مع تطبيق أفضل الممارسات (Best Practices) في الأمان، وإدارة المستخدمين، والتعامل مع المدفوعات.

الميزات الأساسية (Core Features)

1. نظام المصادقة والأمان (Authentication & Security)

إنشاء حساب وتسجيل دخول: باستخدام (JWT) Tokens.

إعادة تعيين كلمة المرور: نظام كامل لإرسال رابط إعادة التعيين عبر الإيميل (nodemailer) وتحديث كلمة المرور.

تأمين الـ API:

Helmet: لإضافة هيدرز الأمان الأساسية.

Express Rate Limit: للحماية من هجمات (Brute Force).

XSS-Clean & HPP: للحماية من هجمات (XSS) و (Parameter Pollution).

التحقق من المدخلات (Validation): استخدام express-validator لتنظيف وتأمين جميع مدخلات المستخدمين.

2. إدارة المستخدمين والأدوار (Roles Management)

المشروع يدعم 3 أدوار مختلفة بصلاحيات منفصلة:

User (المستخدم/المريض): يمكنه البحث عن الأطباء، الحجز، الدفع، إلغاء المواعيد، وكتابة المراجعات.

Doctor (الطبيب): له لوحة تحكم خاصة لإدارة مواعيده المتاحة، تأكيد/إلغاء الحجوزات، وتحديث ملفه الشخصي.

Admin (الآدمن): له صلاحيات كاملة لإضافة الأطباء، إدارة المستخدمين، والموافقة على الحسابات.

3. نظام الحجز وإدارة المواعيد (Booking & Availability)

إدارة متقدمة للمواعيد (Slots): يمكن للأطباء تحديد أيام وساعات العمل الخاصة بهم ومدة الكشف (مثل: 30 دقيقة).

منطق حجز ذكي: النظام يتحقق تلقائياً إذا كان الموعد المطلوب متاحاً ضمن (Slots) الطبيب، وأنه لم يُحجز من قبل.

إدارة حالة الحجز:

يمكن للمستخدم إلغاء حجزه (بشرط أن يكون قبل 24 ساعة من الموعد).

يمكن للطبيب تأكيد (Confirmed) أو إلغاء (CancelledByDoctor) الحجوزات.

4. نظام الدفع الإلكتروني (Payment Integration)

التكامل مع Stripe:

عند طلب الحجز، يتم إنشاء "جلسة دفع" (Stripe Checkout Session) آمنة.

Webhooks: يستخدم النظام (Stripe Webhooks) للاستماع لعمليات الدفع الناجحة، وعندها فقط يتم إنشاء الحجز في قاعدة البيانات وتأكيده.

5. البحث والفلترة (Advanced Search)

APIFeatures:

فلترة (Filtering): البحث بالتخصص، أو التقييم (أعلى من 4.5 مثلاً).

ترتيب (Sorting): الترتيب حسب السعر، أو أعلى تقييم.

تقسيم الصفحات (Pagination): عرض 10 نتائج فقط في كل صفحة.

6. نظام المراجعات والتقييم (Review System)

يمكن للمستخدمين (المرضى) إضافة تقييم (Rating) وتعليق (Comment) للطبيب.

حساب تلقائي: يتم تحديث متوسط تقييم الطبيب (averageRating) وعدد المراجعات (numberOfReviews) تلقائياً في ملف الطبيب بعد كل مراجعة جديدة.

7. لوحات التحكم (Dashboards)

تعديل الملف الشخصي: يمكن للمستخدمين والأطباء تحديث بياناتهم (الاسم، الهاتف) ورفع صورة شخصية (باستخدام multer).

لوحة تحكم الطبيب: مسارات API خاصة بالطبيب لجلب حجوزاته (/my-bookings) وتحديث مواعيده (/my-availability).

المكدس التقني (Tech Stack)

Node.js: بيئة تشغيل الـ JavaScript.

Express.js: إطار العمل لبناء الـ API.

MongoDB: قاعدة البيانات (NoSQL).

Mongoose: لربط التطبيق بقاعدة البيانات.

JSON Web Tokens (JWT): لإدارة جلسات المستخدمين (Authentication).

Stripe: لمعالجة عمليات الدفع.

Nodemailer: لإرسال الإيميلات (مثل إعادة تعيين كلمة المرور).

Multer: للتعامل مع رفع الملفات (الصور والفيديوهات).

bcryptjs: لتشفير كلمات المرور.

express-validator: للتحقق من صحة المدخلات.

هيكل المشروع (Project Structure)

/
├── config/             # ملفات الإعداد (مثل database.js)
├── controllers/        # المنطق (Logic) الخاص بكل مسار
├── middlewares/        # الوسيط (مثل auth, error, upload)
├── models/             # هياكل قاعدة البيانات (Mongoose Schemas)
├── routes/             # مسارات الـ API (Endpoints)
├── utils/              # الأدوات المساعدة (Email, AppError, APIFeatures)
├── validators/         # ملفات التحقق من المدخلات (مفصولة)
├── uploads/            # (مجلد لحفظ الملفات المرفوعة)
├── .env                # ملف المتغيرات السرية
├── app.js              # الملف الرئيسي لتشغيل التطبيق
└── package.json        # الاعتماديات (Dependencies)
